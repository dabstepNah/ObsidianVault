`Dockerfile` -  это декларативный сценарий сборки контейнерного образа 
Он описывает: 
- базовый образ (`FROM`)
- установку зависимостей (`RUN`)
- копирование кода (`COPY, ADD`)
- переменные окружения (`ENV`)
- рабочую директорию (`WORKDIR`)
- команду запуска(`CMD, ENTRYPOINT`)
По сути `Dockerfile`  = воспроизводимая сборка runtime-окружения приложения.

**Синтаксис:**
1) `FROM<image>[:tag] [AS name]`
	примеры: `FROM ubuntu:22.04 ` , `FROM maven:3.9-eclipse-temurin-17 AS builder`
	применение:
	-  каждый `FROM` начинает  новую stage
	- В `multi-stage` может быть несколько `FROM`
2) `WORKDIR /path`
	- Установка текущей директории или ее создание если ее нет (как `mkdir` - в линукс системах)
	- Все операции `RUN, COPY, CMD` - работают относительно этой директории
3) `COPY <src> <dest>`
	`COPY ["src", "dest"]` - `JSON`-форма
	примеры:` COPY pom.xml .`
	`COPY src ./src`
	`COPY --from=builder /app/app.jar app.jar	`
	применение:
	-  копирование файлов с хоста в образ 
	- или из другой стадии `from`
	- `<src>` - пусть относительно `build`- контекста
	- `<dest>` - путь внутри контейнера 
4)  `RUN <shell command>`
	`RUN ["executable", "param1", "param2"]` - `exec` - форма
	пример: `RUN apt update && apt install -y curl`
	применение:
	- выполнение команд во время сборки 
	- создание нового слоя (каждая новая `RUN` = новый слой )
	- все, что создает `RUN` сохраняется в образ 
5) `CMD ["executable", "param1"]`
	`CMD command param`
	пример:
	`CMD ["java", "-jar", "app.jar"]`
	применение: 
	- задачи командый по умолчанию при запуске контейнера 
	- может быть переопределена при `docker run`
6) `ENTRYPOINT ["executable", "param"]`
	применение: 
	- задача основной команды контейнера 
	- не переопределяется полностью через `docker run`
7) `EXPOSE 8080`
	применение: 
	- документация порта
	- скрытие порта (не открывает порт наружу)
	- для публикации нужен `docker run -p 8080:8080`
