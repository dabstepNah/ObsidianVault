1) **`ping`** — проверить доступность хоста по сети
   - `ping example.com` — проверить доступность хоста
   - `ping -c 5 google.com` — отправить 5 пакетов
1) **`curl`** — мощная утилита для передачи данных по URL (работа с HTTP/HTTPS) 
   - `curl -I http://example.com` — получить только заголовки ответа
   - `curl -X POST -H "Content-Type: application/json" -d '{"key":"value"}' http://api.endpoint.com` — POST запрос с JSON
   - `curl -O http://example.com/file.zip` — скачать файл
3) **`wget`** — не интерактивное скачивание файлов из сети
   - `wget http://example.com/file.iso` — скачать файл
   - `wget -c http://example.com/bigfile.zip` — продолжить прерванную загрузку
1) **`ss / netstat`** — информация о сетевых соединениях и портах (диагностика: "почему мой сервер не стартует?")
   - `ss -tulpn` — показать все слушающие TCP/UDP порты и процессы
   - `ss -t` — показать установленные TCP соединения
5) **`nc (netcat)`** — утилита для работы с сетью
   - `nc -zv hostname 22` — проверить, открыт ли порт 22 на удаленном хосте
   - `nc -l 8080` — слушать на порту 8080
6) **`dig / nslookup`** — DNS-запросы
   - `dig example.com` — получить всю DNS-информацию о домене
   - `nslookup example.com` — выполнить DNS-запрос
7) **`ssh (Secure Shell)`** — безопасное подключение к удаленным серверам
   - `ssh user@hostname` — подключение к серверу
   - `ssh -i /path/to/private_key.pem user@hostname` — подключение с использованием ключа
   - `ssh -p 2222 user@hostname` — подключение к нестандартному порту
8) **`scp (Secure Copy)`** — безопасное копирование файлов по SSH
   - `scp file.txt user@remotehost:/path/` — скопировать файл на удаленный сервер
   - `scp user@remotehost:/path/file.txt ./` — скопировать файл с удаленного сервера
